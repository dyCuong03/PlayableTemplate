# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Cocos Creator **3.8.7** playable-ad template (TypeScript). The repo is opened as a Cocos project — there is no top-level npm workspace; `package.json` at the root is the Cocos project manifest, not a Node package.

## Build & Run

- Open the project: `CocosCreator --project .`
- Build playable HTML5 bundle (output → `build/web-mobile`):
  `CocosCreator --project . --build "platform=html5;debug=false"`
- Quick runtime check: Cocos Creator simulator (`Ctrl+P`).
- TypeScript compile check for any directory with its own `tsconfig.json`: `npx tsc -b` (use `-w` to watch). The root `tsconfig.json` extends `./temp/tsconfig.cocos.json`, which is generated by the editor — opening the project in Cocos Creator at least once is required before `tsc` works.

### Super HTML extension

The editor extension at [extensions/super-html](extensions/super-html) is what bundles the build into a single HTML playable. It compiles independently:

```
cd extensions/super-html
npm install          # once
npx tsc -b           # production compile → dist/
npx tsc -w           # watch mode while iterating
```

It is a third-party Cocos store extension (v5.2.2, see [extensions/super-html/package.json](extensions/super-html/package.json)); dependencies include `cheerio`, `jszip`, `clean-css`, `uglify-js`, `mime-types`. It's invoked via the Cocos menu (`open-panel`) and as a build contribution (`./dist/3x/builder.js`).

## Generated / non-source directories

Do not edit or commit: `build/`, `library/`, `temp/`, `profiles/`, `extensions/super-html/dist/`, and Cocos `.meta` sidecars are generated alongside their owning file.

## Source layout

- [assets/scenes/main-scene.scene](assets/scenes/main-scene.scene) — the playable's only scene.
- [assets/configs/constant.ts](assets/configs/constant.ts) — runtime constants. **`constant.TRACKING`** (PLAYABLE_ID, PACKAGE_NAME, BASE_URL, MAX_TRACKING_DURATION_SEC, ENV) drives the analytics pixel; **`constant.STORE_LINK`** sets the install CTAs; **`AUDIO_NAME` / `EFFECT_NAME`** are string keys consumed by audio/FX systems. Most playable retargeting starts by editing this file.
- [assets/plugins/playable-foundation](assets/plugins/playable-foundation) — intended home for shared framework code (signal bus, lifecycle, object pool per [AGENTS.md](AGENTS.md)). Currently empty in this branch; add new framework modules under this path so the bundler picks them up.
- `assets/prefabs/`, `assets/resources/` — reusable nodes and runtime-loaded data. Reference assets via `db://` aliases and mirror this folder hierarchy when adding modules.

## Conventions

- TypeScript, 4-space indent, semicolons.
- Cocos components: declare with `@ccclass` using PascalCase class names (e.g., `GameInit`); methods and properties in camelCase.
- Before submitting, run `npx tsc -b` in any directory that owns a `tsconfig.json`.
- Commit style: short imperative subject (e.g., `Implement signal bus system`); reference affected scenes/prefabs in the body.
- Test exploratory scripts may be named `test*.ts` while iterating, but rename to the target feature before merging.

## Testing

There is no automated test runner. Verify gameplay changes either in the Cocos Creator simulator or by building the HTML5 bundle and opening `build/web-mobile/index.html`. Document manual repro steps in the PR; attach the built HTML when feasible.

## Note on TRACKING.md

[TRACKING.md](TRACKING.md) documents a Unity/C# quest-tracking system (`Assets/Scripts/QuestModule/...`, `signalBus.Fire(...)`, `csharp`) that does **not** exist in this Cocos/TypeScript repo. Treat it as carry-over reference material from another project — do not assume any of its files, types, or signal names are present here. The actual analytics in this codebase is the pixel ping configured via `constant.TRACKING`.
